PHASE 1: Download & Install Everything

Step 1

Install Cursor IDE [DO THIS FIRST!]

Cursor is an AI-powered code editor with a built-in terminal. We'll use Cursor for everything!

[SCREENSHOT: Cursor IDE homepage at cursor.com]

Official Website: https://cursor.com

Why Cursor?

  • AI-powered code completion and suggestions
  • Built-in terminal (we'll use this for all commands!)
  • Works with all VS Code extensions
  • Available on Windows, macOS, and Linux
  1. Go to https://cursor.com/download
  2. Click "Download for Windows"
  3. Run the downloaded .exe file
  4. Follow the installation wizard (default settings are fine)
  5. Launch Cursor

[SCREENSHOT: Cursor installation wizard on Windows]

  1. Go to https://cursor.com/download
  2. Click "Download for macOS"
  3. Open the downloaded .dmg file
  4. Drag Cursor to your Applications folder
  5. Launch Cursor from Applications

[SCREENSHOT: Cursor .dmg installation on macOS]

  1. Go to https://cursor.com/download
  2. Download the AppImage for your architecture (x86_64 for most systems)
  3. Make it executable:
chmod +x Cursor-*.AppImage
  1. Run it:
./Cursor-*.AppImage

System Requirements:

  • At least 4 GB RAM
  • 2 GB available disk space

Open Cursor's Terminal

Once Cursor is open, open the built-in terminal:

Windows/Linux: Ctrl + ` (backtick)
macOS: Cmd + ` (backtick)

OR: Go to Terminal ‚Üí New Terminal in the menu

[SCREENSHOT: Cursor IDE with terminal panel open at bottom]

You'll use this terminal for all the commands below! (Or use your OS terminal if you prefer.)
Step 2

Install Git

Git is version control software that lets you download and manage code.

  1. Download Git from: https://git-scm.com/download/win
  2. Run the installer (accept default options)

[SCREENSHOT: Git for Windows installer]

  1. Verify in Cursor's terminal:

    • You should see something like git version 2.x.x

Step 3

Install Miniforge (Python)

Miniforge is a lightweight conda installer that will manage Python and all scientific packages.

  1. Download Miniforge: Miniforge3-Windows-x86_64.exe
  2. Run the installer
  3. IMPORTANT: Check the box "Add Miniforge to my PATH environment variable"

[SCREENSHOT: Miniforge installer with PATH checkbox highlighted]

  1. Complete installation
  2. Close and reopen Cursor (so it picks up the new PATH)
  3. Verify in Cursor's terminal:
conda --version

You should see conda 24.x.x or similar

[SCREENSHOT: Cursor terminal showing 'conda --version' output]

In Cursor's terminal, run:

curl -L -O "https://github.com/conda-forge/miniforge/releases/latest/download/Miniforge3-MacOSX-$(uname -m).sh"
bash Miniforge3-MacOSX-$(uname -m).sh
  • Press Enter to continue
  • Type "yes" when asked
  • Close and reopen Cursor's terminal
  • Verify: conda --version

[SCREENSHOT: Miniforge installation in terminal]

In Cursor's terminal, run:

curl -L -O "https://github.com/conda-forge/miniforge/releases/latest/download/Miniforge3-Linux-x86_64.sh"
bash Miniforge3-Linux-x86_64.sh
# Follow prompts
# Close and reopen terminal
conda --version
Note: If you already have Conda or Anaconda installed, you can skip Miniforge.

Step 7

Create Python Environment

Now we'll create a dedicated Python environment with all required packages using Cursor's terminal.

‚ö†Ô∏è

CRITICAL: Environment Name

Your environment MUST be named:

numbers_eeg_source

This ensures compatibility with all pipeline scripts!

Create the Environment

In Cursor's terminal, make sure you're in the project directory, then run:

Method 1: Using environment.yml (Easiest)
# Make sure you're in the project root pwd # Check your location first # Create environment from file conda env create -f environment.yml # This will take 5-10 minutes to download and install all packages

[SCREENSHOT: Cursor terminal showing conda env create in progress]

Method 2: Manual Installation
# Create the environment with Python 3.11 conda create -n numbers_eeg_source python=3.11 -y # Activate it conda activate numbers_eeg_source # Install all packages (this will take 5-10 minutes) conda install -c conda-forge mne numpy scipy matplotlib pandas tqdm imageio nibabel nilearn pyvista pyvistaqt pyqt vtk pyyaml statsmodels pingouin seaborn -y

Activate the Environment

In Cursor's terminal:

conda activate numbers_eeg_source

How to tell if it's activated:

  • Your prompt will show (numbers_eeg_source) at the beginning
  • Example: (numbers_eeg_source) C:\Users\YourName>

[SCREENSHOT: Cursor terminal with (numbers_eeg_source) prefix showing environment is active]

Verify Installation

With the environment activated in Cursor's terminal, run these tests:

You should see output like:

Troubleshooting

Git Issues

"Git clone failed with authentication error"

  • The repository may be private. Ask your lab for access permissions.
  • You may need to set up SSH keys or a Personal Access Token (PAT).

Data Issues

"Can't find the data_preprocessed folder on Google Drive"

  • Make sure you're logged into the correct Google account.
  • Ask your lab manager for access to the shared drive.

"Data folder is huge (several GB)!"

  • The full dataset with all preprocessing variants is large.
  • You only need: hpf_1.5_lpf_35_baseline-on/
  • You can delete other preprocessing folders if needed to save space.

Cursor Issues

"Cursor won't open on macOS - 'App is damaged'"

  • This is a security warning.
  • Go to System Preferences ‚Üí Security & Privacy
  • Click "Open Anyway" next to the Cursor warning

"Cursor doesn't show my conda environment in interpreter list"

  • Make sure conda is initialized: conda init
  • Restart Cursor after creating the environment
  • Try entering the full path manually:
    • Ctrl+Shift+P ‚Üí "Python: Select Interpreter" ‚Üí "Enter interpreter path"

Conda/Environment Issues

"conda: command not found" after installing Miniforge

  • You need to restart your terminal/PowerShell
  • Or manually initialize: conda init bash (or conda init zsh for macOS)
  • Then restart terminal

"Package installation failed with conflicts"

  • Try creating a fresh environment:
conda remove -n numbers_eeg_source --all conda env create -f environment.yml
  • If still failing, try Method 2 (manual installation) one package at a time

"Python says 'No module named mne' even after installation"

  • Make sure you activated the environment: conda activate numbers_eeg_source
  • Check that Cursor is using the correct interpreter (Ctrl+Shift+P ‚Üí "Python: Select Interpreter")
  • Verify which Python you're using:
    • macOS/Linux: which python
    • Windows: where python

"PyVista/VTK installation fails"

  • These packages can be tricky on some systems
  • Try installing separately: conda install -c conda-forge pyvista -y
  • On older Macs, you may need to use pip: pip install pyvista

Shell Script Issues

"Permission denied when running shell scripts"

  • Make the script executable: chmod +x new_user/run_analysis.sh
